<objective>
Create the initial Rust `aliasman` CLI package and reusable alias store foundation. The completed work must provide `aliasman --help` and `aliasman --version`, a full v0.0.1 alias metadata model, TOML serialization helpers, deterministic managed alias rendering, atomic file writes, strict alias-name validation, protected-name force handling, and the Phase 1 unit/tempdir test baseline.
</objective>

<requirements>
- **FND-01:** User can install and run an `aliasman` Rust CLI binary with global help and per-command help.
- **FND-02:** User alias metadata is stored in a canonical aliasman data file separate from generated shell output.
- **FND-03:** aliasman writes managed alias output atomically and can regenerate it from canonical metadata.
- **FND-04:** aliasman rejects invalid alias names and warns before protected command names can be shadowed.
</requirements>

<truths>
- Phase 1 exposes only `aliasman --help` and `aliasman --version`; do not add future command stubs.
- The canonical alias record includes `name`, `command`, optional `description`, `tags`, `shell`, `source`, `created_at`, and `updated_at`.
- Generated alias output targets fixed `~/.aliases` conceptually, but implementation helpers accept explicit paths for tests and later shell integration.
- Alias-name validation uses `[A-Za-z_][A-Za-z0-9_-]*`.
- Protected names require an explicit force path before write validation passes.
- Tests must not touch real home-directory shell files.
</truths>

<threat_model>
## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Atomic write helper corrupts a target file on partial writes | high | Implement same-directory temporary file writes with `tempfile`, flush before persist, and verify overwrite behavior in tempdir integration tests. |
| Tests accidentally write to the user's real `~/.aliases` or shell config | high | All filesystem tests must use tempdir paths passed explicitly into store helpers; no test may resolve or write the real home directory. |
| Alias validation permits dangerous command shadowing by default | medium | Implement protected-name detection and require `force = true` in `validate_alias_name_for_write`. |
| Rendered alias output is malformed when commands contain single quotes | medium | Implement shell single-quote escaping and cover it with a deterministic rendering test. |
</threat_model>

- Create the parent directory if it does not exist.
- Create the temporary file in the same directory as the target.
- Write all contents.
- Flush the temporary file.
- Persist the temporary file over the target path.

`write_managed_aliases` should render the store and call `write_atomic`.
