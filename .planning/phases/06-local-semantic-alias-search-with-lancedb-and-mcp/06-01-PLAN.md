<objective>
Implement local semantic alias search using Ollama embeddings and LanceDB, keep the index fresh after alias mutations, expose `aliasman search "query"` and `aliasman search reindex`, and expose one read-only MCP `alias_search` tool for Claude.
</objective>

<requirements>
- **SEM-01:** User aliases are embedded locally and indexed in a LanceDB-backed vector store.
- **SEM-02:** aliasman can refresh the semantic alias index when aliases are added, updated, deleted, imported, or suggested.
- **SEM-03:** User can search aliases semantically from the CLI using natural-language queries.
- **SEM-04:** Claude can search aliases semantically through a local MCP tool/server instead of relying only on hook-injected context.
- **SEM-05:** Semantic search runs locally by default and does not send alias commands or metadata to remote embedding services unless the user explicitly configures that later.
</requirements>

<truths>
- Embedding provider is local Ollama for v0.0.1.
- No remote embedding service is used by default.
- LanceDB is the local vector store.
- Index refresh happens on add/update/delete/import/suggestion-apply and can be repaired with `aliasman search reindex`.
- Primary CLI UX is `aliasman search "query"` with `Alias`, `Command`, `Score`, `Reason`, default top 5, and optional `--limit`.
- MCP exposes one read-only `alias_search(query, limit?)` tool.
- Ollama, LanceDB, or MCP search failures warn and fall back to lexical search where possible.
</truths>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| Alias commands or metadata are sent to a remote embedding service | high | Default embedding provider is local Ollama at localhost; no remote provider is implemented in Phase 6. Tests assert provider URLs are local/injected. |
| Semantic index becomes stale after alias mutations | medium | Hook index refresh into successful add/update/delete/import/suggestion-apply flows and provide `aliasman search reindex`. |
| Ollama or LanceDB failure breaks alias workflows | medium | Search paths warn and fallback to lexical search; mutation paths should not fail the alias write solely because index refresh failed unless explicitly configured later. |
| MCP tool mutates user aliases or shell files | medium | MCP `alias_search` is read-only and only returns search results. |
</threat_model>
