<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| pack file (TOML) → install_pack() | Untrusted pack content crosses into alias store; safety scanner validates before write |
| URL download → parse_pack_file | Remote content parsed as TOML; malicious URLs could serve dangerous aliases |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation |
|-----------|----------|-----------|-------------|------------|
| T-09-01 | Spoofing | download_pack() | mitigate | parse_pack_file validates TOML structure; create_install_preview shows all aliases before install |
| T-09-02 | Tampering | pack TOML aliases | mitigate | scan_pack_safety() blocks command substitution, pipe-to-shell, destructive commands, network access, sensitive file writes |
| T-09-03 | Information Disclosure | InstallPreview::display() | accept | Preview only shows alias names and commands; no sensitive data exposed |
| T-09-04 | Denial of Service | install_pack() two-phase | mitigate | Phase 1 validates all aliases before any write; atomic apply prevents partial state |
| T-09-05 | Elevation of Privilege | --force flag | accept | User explicitly opts in; documented as override for safety warnings only |
</threat_model>
