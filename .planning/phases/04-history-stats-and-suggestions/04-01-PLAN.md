<objective>
Add shell-history analytics and safe alias suggestions. Users can inspect frequent commands, use richer verbose stats, view generated alias suggestions with reasons, see risky-command warnings, and explicitly apply a suggestion through existing CRUD validation.
</objective>

<requirements>
- **HST-01:** User can view command frequency statistics from zsh or bash history.
- **HST-02:** aliasman correctly parses zsh extended history format.
- **HST-03:** aliasman suggests short aliases for frequent long commands.
- **HST-04:** User must explicitly approve a history-derived suggestion before it becomes an alias.
- **HST-05:** aliasman flags risky history-derived commands that contain command substitution or other shell metacharacters.
</requirements>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| History suggestion auto-creates unsafe alias | high | `suggest` is display-only unless an explicit accept/apply path is provided and routed through CRUD validation. |
| zsh extended history timestamps are treated as commands | medium | Dedicated zsh extended parser with fixtures. |
| Large history file causes slow or memory-heavy command | medium | Use streaming or bounded line processing. |
</threat_model>

<truths>
- `stats` and `suggest` auto-detect history files but also support `--history-file <path>`.
- Default `stats` output is a compact `Count`/`Command` table.
- A verbose stats mode includes percentages and grouping by executable/tool.
- `suggest` generates alias names automatically and shows command plus reason.
- Risky suggestions are marked `Review carefully` and are never auto-applied.
- `suggest` is display-only by default; explicit apply behavior routes through CRUD validation.
</truths>
