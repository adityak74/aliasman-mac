<objective>
Implement Claude Code hook installation and hook execution. The user can install an aliasman `SessionStart` hook, existing Claude settings are preserved, and the hook runner emits only relevant alias context within a token budget.
</objective>

<requirements>
- **CLD-01:** User can install an aliasman Claude Code `SessionStart` hook without overwriting existing Claude settings.
- **CLD-02:** The Claude hook command emits valid hook JSON containing alias context when relevant aliases exist.
- **CLD-03:** The Claude hook filters aliases by current project context instead of injecting the entire alias list.
- **CLD-04:** The Claude hook respects a configurable token budget for injected context.
- **CLD-05:** The Claude hook exits cleanly with no noisy output when no alias context should be injected.
</requirements>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| Hook install overwrites existing Claude settings | high | Parse/merge JSON and write atomically; tests preserve unrelated keys and hooks. |
| Hook dumps all aliases and wastes context | medium | Score/filter aliases by project context and enforce token budget. |
| Hook crashes or emits malformed JSON | medium | Return no output on no-op/error paths and test JSON output shape. |
</threat_model>
